# IA na prática: Acelerando o desenvolvimento e garantindo a qualidade com GitHub Copilot e GitHub Actions

Este projeto foi desenvolvido como atividade acadêmica com o objetivo de demonstrar como ferramentas de Inteligência Artificial podem acelerar o desenvolvimento de software sem comprometer a qualidade. Para isso, foi utilizado o GitHub Copilot para auxiliar na geração de código e testes automatizados, enquanto o GitHub Actions foi empregado para automatizar a execução desses testes sempre que novas alterações são enviadas ao repositório.

# Análise do Problema

### Desenvolvimento lento
Os desenvolvedores gastam grande parte do tempo criando estruturas básicas do sistema, como classes, componentes e endpoints, reduzindo o tempo disponível para implementar funcionalidades realmente importantes.

### Poucos testes
Como a prioridade é entregar rapidamente novas funcionalidades, os testes automatizados acabam sendo deixados para depois ou sequer são criados, aumentando o risco de falhas.

### Bugs
Muitos problemas só são descobertos durante o processo de testes manuais ou após a publicação do sistema, tornando sua correção mais cara e demorada.

### Código inconsistente
Cada desenvolvedor implementa soluções de maneira diferente, dificultando a manutenção da aplicação.

# O papel da Inteligência Artificial no Desenvolvimento de Software

O GitHub Copilot funciona como um assistente inteligente para programação. Utilizando modelos de Inteligência Artificial, ele sugere códigos completos, funções, comentários e testes automatizados enquanto o desenvolvedor escreve.

### Benefícios
* Aumenta a produtividade
* Reduz tarefas repetitivas
* Auxilia desenvolvedores iniciantes
* Gera testes rapidamente
* Ajuda na documentação do código

# GitHub Actions

O GitHub Actions é uma plataforma de automação integrada ao GitHub que permite executar tarefas automaticamente após eventos como um push ou pull request.

* Instalação automática das dependências
* Execução dos testes
* Validação do projeto
* Integração contínua (CI)
* Entrega contínua (CD)

# Introdução ao CI/CD

CI significa Continuous Integration, é quando todo código enviado é testado automaticamente.

CD significa Continuous Delivery, é a preparação do software para ser entregue de forma rápida e segura.

# Caso Real

Um exemplo bastante conhecido é a GitHub, empresa responsável pelo próprio GitHub e pelo GitHub Copilot.

A GitHub divulgou estudos mostrando que desenvolvedores utilizando o GitHub Copilot conseguem concluir tarefas mais rapidamente e dedicar mais tempo à resolução de problemas complexos, reduzindo o tempo gasto com códigos repetitivos. Esse resultado demonstra como a IA pode aumentar a produtividade sem substituir o papel do desenvolvedor.

# Conclusão

* IA não substitui o programador
* Ela auxilia nas tarefas repetitivas
* Aumenta a produtividade
* Melhora a qualidade
* Reduz bugs
* Acelera o desenvolvimento

A utilização conjunta do GitHub Copilot e do GitHub Actions demonstra que é possível desenvolver software de forma mais rápida e segura. Enquanto o Copilot acelera a escrita de código e testes, o GitHub Actions garante que essas alterações sejam verificadas automaticamente, contribuindo para um processo de desenvolvimento mais eficiente e confiável.
